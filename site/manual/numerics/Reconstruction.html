Solution Reconstruction

This is an important topic. See this:

\[q_j(x_{i+1/2}) \approx q_j(x_i) + \frac{h}{2} \delta q_j\]

In the case of higher order schemes, it is necessary to give a good approximation of the interface values, from which the fluxes are calculated. If the reconstruction is neglected i. e. the cell average is considered to be the interface value, we end up with the simple first order Godunov method. (In FDM this would be the Courant-Isaacson-Rees method).

Higher order schemes use some kind of interpolant to calclutae the interfaces values at the two s ide of the interface, which is later used for either solving the (approximate) Riemann problem (MUSCL methods), or other techniques are applied (e.g. Rusanov fluxes). The most common interpolants are the piecewise linear and piecewise parabolic reconstruction methods. The reconstruction method may apply so-called flux @ref Limiters which modify the exact interpolant to prevent the introduction of new extrema into the solution, thus preventing Gibbs phenomenon.

group Reconstruction

Solution reconstruction aims to reproduce the interface values to a given precision.

Reconstruction is defined seperately from the slope limiters, allowing larger modularity.

The base concept here is, that the classes derived from ReconstructionBase can be used to derive the left and right interface states relative to the cell center. This is important implementation detail, that at the current states every interface is calculated twice. The need of separate storage per interface is still a question. This would need further syncronisasions among threads and the memory access can be a bottleneck. Thus, at least for the reference implementation this need this way and FluxOperator class can be different.

template<QuantityType Q>
class ReconstructionBase
#include <advection/flux_operators.hpp>

Abstract interface for solution reconstruction methods.

Technically it is just a callable that provides left and right interface states.

Subclassed by PCM< Q, Dir, Limiter >, PLM< Q, Dir, Limiter >

class PCM : public ReconstructionBase<Q>
#include <advection/flux_operators.hpp>

Piecevise Constant Reconstruction.

The simple procidure gives the interface value of \( q \) as

\[ q_{i+1/2}^- = q_i \qquad \text{and} \qquad q_{i+1/2}^- = q_{i+1} \]

Note

the Limiter is not used here and will be removed later probably.

Template Parameters:

  • Q – Type of the statevector

  • Dir – Constant Direction enum. Determining the operator direction compile time

  • Limiter – LimiterBase child object that defines the flux limiter.
